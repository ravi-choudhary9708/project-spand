(cid:0) Overall Description
2.1 Product Perspective
The proposed system is a Quantum-Safe Cryptographic Inventory Scanner designed
to identify and analyze cryptographic implementations used in public-facing applications
such as web servers, APIs, and TLS-based services.
The system evaluates cryptographic implementations used in services such as web
servers, APIs, email servers, file transfer services, and VPN endpoints. It extracts
cryptographic configurations, evaluates their resistance to quantum attacks, and
generates a Cryptographic Bill of Materials (CBOM) to assist organizations in preparing
for the transition to Post-Quantum Cryptography (PQC)

2.2 Product Functions
The Quantum-Proof Systems Scanner provides the following core functionalities.
1.Asset Input and Discovery
The system accepts a manual list of assets provided by the organization, such as domains, URLs, or IP
addresses. In addition to manual input, the system performs automated asset discovery to identify additional
publicly exposed services using:
Certificate-associated domains
subdomains
Mail and DNS infrastructure hints
Both manually provided and automatically discovered assets are merged to ensure that no exposed system is missed.
2. Asset Validation and Scope Enforcement
The system verifies that all assets fall within the authorized scanning scope.
This ensures that the scanner operates within authorized security testing boundaries.
3.DNS Resolution
The system resolves all discovered assets into IP addresses using DNS queries.
The module performs:
A record resolution, AAAA record resolution, IP deduplication
=== PAGE 6 ===
4.CDN and WAF Detection
The system identifies whether the target infrastructure is protected by a Content Delivery Network (CDN)
or Web Application Firewall (WAF).
If CDN protection is detected, the system marks the TLS configuration as Edge TLS and attempts to
identify alternative origins through passive discovery.
5.Service Discovery
The system performs network scanning to identify exposed services on the discovered assets.
Service discovery is conducted using Nmap, targeting commonly used secure service ports
The discovered open ports and services are recorded in the asset database.
6.Protocol Detection
The system analyzes each discovered service to determine the communication protocol used.
Supported protocol detection includes:
HTTPS and web APIs
SMTP and STARTTLS email servers
IMAP and POP3 email retrieval services
FTPS file transfer services
SSH remote access services
VPN endpoints
Each asset is classified according to its service category, such as web, API, mail, file transfer, or VPN.
7.Protocol-Aware Cryptographic Scanning
The system performs protocol-specific cryptographic analysis using specialized scanning modules.
Examples include:
HTTPS scanning using TestSSL and SSLyze
SMTP STARTTLS cryptographic inspection
IMAP and POP3 TLS configuration analysis
FTPS cipher and TLS inspection
SSH cryptographic algorithm auditing
VPN TLS configuration analysis
These modules extract encryption parameters used by each protocol.
8.Cryptographic Data Extraction
The system parses scan results to extract detailed cryptographic parameters, including:
TLS protocol versions
Cipher suites
Key exchange algorithms
Certificate chains
Key sizes
Signature algorithms
This information forms the basis of the organization’s cryptographic inventory.
9.Quantum Risk Assessment
The system evaluates extracted cryptographic algorithms to determine their vulnerability to quantum
computing attacks.
Algorithms such as:
RSA
Elliptic Curve Cryptography (ECC)
Diffie-Hellman
are flagged as quantum vulnerable.
The system calculates a Harvest-Now-Decrypt-Later (HNDL) risk score for each asset using multiple
factors such as algorithm strength, key size, and certificate lifetime.
=== PAGE 7 ===
10.Compliance Mapping
The system maps discovered cryptographic configurations to relevant cybersecurity frameworks and
standards.
Findings are correlated with:
RBI cybersecurity directions
NIST Post-Quantum Cryptography guidance
NIST IR 8547 PQC readiness model
CERT-In security advisories
This enables organizations to assess their regulatory compliance posture.
11.Cryptographic Correlation and Inventory Management
The system aggregates cryptographic findings across all scanned assets to create an organization-wide
cryptographic inventory.
Examples include identifying:
Number of systems using RSA-2048
Services using outdated TLS versions
Assets requiring PQC migration
This provides visibility into cryptographic usage across the entire organization.
12.Cryptographic Bill of Materials (CBOM) Generation
The system generates a structured Cryptographic Bill of Materials (CBOM).
The CBOM lists all cryptographic components used by each asset, including:
Protocol type
Encryption algorithms
Key sizes
Certificates
Risk classification
The CBOM is exported using the CycloneDX standard format.
13.Scan Comparison and Change Detection
The system compares current scan results with historical scans stored in the database.
This enables detection of:
Newly introduced vulnerabilities
Cryptographic upgrades
Security regressions
This feature allows organizations to track changes in their cryptographic posture over time.
14. PQC Readiness Assessment
The system evaluates the organization’s readiness for Post-Quantum Cryptography adoption.
Each asset receives a readiness classification such as:
Quantum Safe
Partially Safe
Vulnerable
Critical Risk
The system also calculates an organization-level PQC readiness score.
15.Reporting and Visualization
The system provides multiple reporting and visualization capabilities.
Available outputs include:
Interactive security dashboard
PDF and HTML security reports
CBOM export files
Compliance reports
Email and webhook alerts
These reports provide actionable insights for security teams and auditors.
16.Continuous Monitoring and Automated Rescanning

3.3 System Features
1. Asset Discovery and Management
The system automatically identifies public-facing assets belonging to an organization. It combines manually provided asset
lists with automated discovery techniques to ensure comprehensive asset coverage.

accept asset lists provided by the organization
perform automated asset discovery using subdomain enumeration
query Certificate Transparency logs to identify domains associated with certificates
analyze DNS records such as MX and SPF
merge and deduplicate discovered assets

The system shall:

analyze key exchange mechanisms and signature algorithms
